# Nex v2.1.0 Release Notes

Release date: 2026-03-02

## Summary

v2.1.0 improves runtime reliability and memory behavior for large discovery roots, adds structured diagnostics for support/perf automation, and makes most config changes apply immediately without restart.

This release intentionally defers icon-pipeline changes from the original plan scope.

## Highlights

1. Structured runtime status JSON
- Added `--status-json` command for machine-readable diagnostics.
- Includes runtime state, memory snapshot data, icon-cache metrics, config reload line metadata, and query latency summaries.

2. Live config apply for most settings
- Runtime now hot-applies safe config edits (result limits, mode/toggles, clipboard/plugin/web settings, memory tuning).
- Discovery provider settings now reconfigure in-process and trigger background reindex automatically.
- `hotkey` and `index_db_path` remain restart-required.

3. Index budget controls (memory envelope)
- New config keys:
- `index_max_items_total` (default `120000`)
- `index_max_items_per_root` (default `40000`)
- `index_max_items_per_query_seed` (default `5000`)
- Discovery now enforces total and per-root caps in both filesystem and Windows Search paths.
- Cache strategy now keeps app items fully hot while compacting file/folder cache for lower steady-state memory.

4. Runtime maintenance action
- Added command action: `Trim Memory Now`.
- Clears runtime overlay caches and records a memory snapshot event for diagnostics.

5. Diagnostics tooling
- Added script: `scripts/windows/profile-memory-and-icons.ps1`.
- Automates broad-root profile setup, runtime restart, status capture, and key log extraction.

## Config and Migration Notes

- Config schema advanced to version `7`.
- Existing configs are migrated automatically to include new index budget keys.
- Template guidance now clarifies which settings are hot-applied and which require restart.

## Operational Notes

- Use `nex.exe --status-json` for support automation and perf dashboards.
- Use `scripts/windows/profile-memory-and-icons.ps1` for reproducible memory profiling.
- Validate with:
- `cargo test -p nex`

## Deferred from This Release

- Icon resolver pipeline redesign and icon-specific release gating were removed from this release scope.
- Current icon behavior remains unchanged in v2.1.0.
